How to build wordpress in docker?

本文将介绍如何在`ubuntu 16.04 LTS`中安装`docker community edition`(`即docker社区版`)并用`docker-compose`这个构建工具来部署`wordpress`内容管理系统!

如果你已经在云服务提供商诸如阿里云,腾讯云,Azure,Amazon,华为云,七牛云,百度云等等购买过云主机，并且操作系统是linux的发行版，那接下来你只需按部就班的来就可以成功运行wordpress了！

安装docker

本文使用docker镜像来自清华大学开源软件镜像站

注意:此镜像只提供 Debian/Ubuntu/Fedora/CentOS/RHEL 的 docker 软件包，非 dockerhub

  • Debian/Ubuntu 用户

以下内容根据 官方文档 修改而来。

如果你过去安装过 docker，先删掉:

sudo apt-get remove docker docker-engine docker.io

首先安装依赖:

sudo apt-get install apt-transport-https ca-certificates curl gnupg2 software-properties-common

根据你的发行版，下面的内容有所不同。你使用的发行版：
信任 Docker 的 GPG 公钥:

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
1.amd64 架构

对于 amd64 架构的计算机，添加软件仓库:

sh sudo add-apt-repository "deb [arch=amd64] https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/ubuntu $(lsb_release -cs) stable"

2.树莓派或其它ARM架构

如果你是树莓派或其它ARM架构计算机，请运行:

sh echo "deb [arch=armhf] https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list
  • 最后安装
sudo apt-get update
sudo apt-get install docker-ce

Docker Compose

为了便于使用，我们需要安装Docker Compose来管理和运行Docker容器，Docker Compose的安装步骤如下：

// 步骤1 - 下载安装文件：
$ sudo curl -L https://github.com/docker/compose/releases/download/1.17.0/docker-compose-`uname -s`-`uname -m` -o /usr/bin/docker-compose

// 步骤2 - 给已下载的安装文件添加执行权限：
$ sudo chmod +x /usr/bin/docker-compose

// 步骤3 - 测试是否安装成功：
$ docker-compose --version
  • 添加一个docker network
网站需要占据80端口，显然，我们的服务器不可能只有一个网站，所以部署一个nginx容器是必须的，让这个nginx来监听80以及443端口，再根据域名转发到对应的网站容器。容器之间的通信是通过network的，所以我们需要先添加一个network：

docker network create nginx-proxy

部署wordpress

创建工作目录，创建docker-compose.yml文件：

cd /var
mkdir myblog && cd myblog
vim docker-compose.yml
在docker-compose.yml输入以下内容


yaml version: '3' services: db: image: mariadb volumes: - db_data:/var/lib/mysql environment: MYSQL_ROOT_PASSWORD: xxxx MYSQL_DATABASE: xxxx MYSQL_USER: xxxx MYSQL_PASSWORD: xxxx wordpress: depends_on: - db image: wordpress volumes: - wp_site:/var/www/html expose: - 80 restart: on-failure:10 environment: WORDPRESS_DB_NAME: xxxx WORDPRESS_TABLE_PREFIX: xxxx WORDPRESS_DB_HOST: db:3306 WORDPRESS_DB_USER: xxxx WORDPRESS_DB_PASSWORD: xxxx nginx-proxy: image: jwilder/nginx-proxy container_name: nginx-proxy restart: always ports: - "80:80" - "443:443" volumes: - /var/run/docker.sock:/tmp/docker.sock:ro - nginx_certs:/etc/nginx/certs:ro volumes: db_data: wp_site: nginx_certs: networks: default: external: name: nginx-proxy

运行

docker compose up -d

然后在你的浏览器里输入http://<您的 CVM IP 地址>:8080
此时，我们的网站就可以访问了，很简单吧？

上传新主题

wordpress的默认的那三个主题我想你肯定不会去选择使用的(什么?你说你就喜欢这种浓浓的复古风，ok，你开心就好!)，所以我们要上传新主题，那么如何上传呢?
  • 解决方案:
    • a. FTP上传
    • b. 直接在docker中修改文件
      • 查看正在运行的容器
        docker ps -a
      • 进入相应的容器 
        docker exec -it (container-id:根据自己实际情况填写) bash
        此时进入的你在docker-compose.yml里面为wordpress设置的volume里面，上面我设置的是/var/www/html
        cd wp-contents/themes
        我选择用wget直接获取挂在CDN上的主题压缩包，当然你也可以用git clone或者curl
        可是当你敲apt-get install wget命令时，不出意外会提示：
        Reading package lists... Done
Building dependency tree
Reading state information... Done
E: Unable to locate package wget
        这时候就需要敲：
        apt-get update
        这个命令的作用是：同步 /etc/apt/sources.list 和 /etc/apt/sources.list.d 中列出的源的索引，这样才能获取到最新的软件包。
        等更新完毕以后再敲命令
        apt-get install wget
        然后直接wget你的主题压缩包！再unzip或者tar -xzvf一下就ok了！

这个时候再进入wordpress后台就会看到你刚才解压后的新主题了，美滋滋！

大功告成！
